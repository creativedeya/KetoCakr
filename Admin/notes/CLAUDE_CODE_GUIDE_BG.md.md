Но има нюанс — за максимална ефективност, препоръчвам **хибриден подход:** - **На български:** обяснения какво искаш, описания на функционалност, въпроси - **На английски:** имена на файлове, променливи, коментари в кода, технически термини **Пример:** ``` Създай Home Screen компонента в app/(tabs)/home/index.tsx. Трябва да зарежда base_recipes от Supabase и да ги показва в grid. Използвай Colors и Theme от constants/ папката. Не използвай hardcoded цветове. ``` Claude Code ще разбере перфектно. Ключовото е да бъдеш **конкретна** — каквъвто и език да използваш. --- ## 🚀 СТЪПКА 1: Настрой CLAUDE.md (еднократно) Създай файл `C:\Dev\KetoCakr\Mobile\CLAUDE.md` с основна информация за проекта. Когато се запълни, Claude Code започва да „забравя" и прави грешки. ### Правило: Никога над 60% контекст ### Как да следиш: - Claude Code показва използвания контекст - Когато стане прекалено дълга сесията → `/compact` - Ако `/compact` не помага → `/clear` и ново задание ### Стратегия „Една фаза = една сесия": ``` Сесия 1: ФАЗА 1 — Tab навигация → /clear Сесия 2: ФАЗА 2 — Home Screen → /clear Сесия 3: ФАЗА 3 — Search Screen → /clear ``` ### Ако контекстът се изчерпа по средата на задача: 1. Кажи на Claude Code: „Запиши какво сме направили до момента и какво остава в PROGRESS.md" 2. `/clear` 3. „Прочети CLAUDE.md и PROGRESS.md. Продължи откъдето спряхме." --- ## 🗂️ СТЪПКА 6: Файлове за проследяване Създай тези файлове в `C:\Dev\KetoCakr\Mobile\`: ### CLAUDE.md (вече го направихме) → Claude Code чете автоматично ### CLAUDE_CODE_TASK.md (вече го направихме) → Детайлно задание с всички фази ### PROGRESS.md (създай при нужда) → Дневник на напредъка ```markdown # KetoCakR Progress ## Завършено - [x] ФАЗА 1: Tab навигация — 23.02.2026 - [ ] ФАЗА 2: Home Screen - [ ] ФАЗА 3: Search Screen ... ## Текущи проблеми - Описание на проблема + в кой файл ## Следваща задача - ФАЗА 2: Home Screen секции 2.1 и 2.2 ``` --- ## 🔄 СТЪПКА 7: Ролята на Claude.ai (мен) vs Claude Code | | Claude.ai (аз) | Claude Code | |---|---|---| | **За какво** | Планиране, дизайн решения, задания, debug на сложни проблеми | Писане и редактиране на код файлове | | **Силна страна** | Виждам „голямата картина", мога да търся в интернет, помня контекста ти | Директен достъп до файловете, може да пише и стартира код | | **Кога** | Преди и след Claude Code сесия | По време на кодене | | **Език** | Български без проблем | Български за инструкции, английски за код | ### Препоръчан цикъл: ``` 1.
