# 🔄 Git Hooks Setup Guide

Автоматична синхронизация на devlog файлове при Git операции.

---

## 📋 Какви hooks са инсталирани?

### 1️⃣ **post-commit** (След всеки commit)
```bash
git commit -m "Your message"
# Автоматично синхронизира devlog файлове
```

**Какво прави:**
- Синхронизира `admin/notes/devlog` ↔ `mobile/notes/devlog` ↔ Obsidian
- Показва статус mensaje
- Не спира commit процеса ако синхронизирането не успее

### 2️⃣ **post-merge** (След всеки merge)
```bash
git merge another-branch
# Автоматично синхронизира devlog файлове
```

**Какво прави:**
- Синхронизира след успешен merge
- Гарантира че всички разклонения имат актуални devlog файлове

### 3️⃣ **pre-commit** (Преди всеки commit)
```bash
git commit -m "Your message"
# Показва предупреждение ако има untracked devlog файлове
```

**Какво прави:**
- Предупреждава ако има нови devlog файлове които не са staged
- Не спира commit-а, само предупрежда

## ⚙️ Управление на hooks

### Деактивирай hooks за един commit

```bash
git commit --no-verify -m "Message"
```

Това **пропуска всички hooks** (pre-commit и post-commit).

### Деактивирай hooks временно

```bash
# Деактивирай всички hooks
git config core.hooksPath ""

# Включи обратно
git config core.hooksPath ".git/hooks"
```

### Редактирай hook

Отвори файла в текстов редактор:

```
C:\Dev\KetoCakr\.git\hooks\post-commit
```

И направи промени директно.

## 🐛 Отстраняване на проблеми

### "Hook не работи"

**Провери:**
1. Дали Node.js е инсталиран: `node --version`
2. Дали npm е инсталиран: `npm --version`
3. Дали hook файлът е executable:
   ```bash
   ls -la .git/hooks/post-commit
   # Трябва да видиш x permission
   ```

### "Hook е бавен"

Синхронизирането може да отнеме време с много файлове. Ако е проблем:

```bash
# Пропусни синхронизирането за този commit
git commit --no-verify -m "Urgent fix"

# Синхронизирай мануално после
npm run sync:devlog
```

### "Hook не синхронизира"

Провери логовете:

```powershell
# Редактирай hook и премахни > /dev/null за да видиш грешки
# Промени тази линия:
npm run sync:devlog > /dev/null 2>&1

# На:
npm run sync:devlog
```

## ❓ Често задавани въпроси

**Q: Ще синхронизира ако нямам интернет?**
A: Синхронизирането е локално (между папките), но Obsidian трябва да съществува.

**Q: Могу ли да редактирам hooks?**
A: Да, редактирай файловете в `.git/hooks/` директорията директно.

**Q: Могу ли да добавя собствен hook?**
A: Да, създай файл в `.git/hooks/` със същото име и направи го executable.

**Q: Какво се случва ако синхронизирането падне?**
A: Commit ще бъде успешен (hooks не блокират), но devlog файлове няма да се синхронизират.
